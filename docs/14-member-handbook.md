# 夜莺社区成员手册（协作版）

> 本手册帮助成员快速了解：社区是什么、在做什么、怎么做、如何参与。

## 1. 快速了解
- 社区总览：[`docs/00-overview.md`](00-overview.md)
- 愿景与原则：[`docs/02-vision-principles.md`](02-vision-principles.md)
- 路线图：[`docs/07-roadmap.md`](07-roadmap.md)

## 2. 加入前准备
- **账号与沟通**：GitHub（代码）、钉钉（会议）、微信（辅助沟通）
- **文档协作**：文档以 `docs/` 为主，必要时整理公开版本再对外发布；如需使用社区 wiki，可向管理员申请权限
- **开发环境**：优先本地开发；如需云上环境，向管理员申请访问（云上默认代码目录为 `$HOME/code`）
- **安全与访问**：准备 SSH 公钥，按密钥方式登录云上机器
- **云资源与资产**：参考 [`docs/20-cloud-assets.md`](20-cloud-assets.md)

## 3. 你可以做什么
- **开发者**：协议、工具链、插件、应用集成
- **研究者**：方法论、评估指标、路线验证
- **创作者**：内容、案例、产品叙事与社区传播
- **生态伙伴**：场景共建与资源协同

## 4. 参与路径（建议）
1) 选择方向：数据层 / 协议层 / AI 层 / 应用层
2) 加入工作组（或发起提案）
3) 完成任务与交付
4) 参与评审与共创展示

## 5. 协作与治理
- 治理原则与流程：[`docs/05-governance.md`](05-governance.md)
- 提案机制：先讨论后执行，重大事项公开评审

## 6. 贡献流程（精简）
- 提交 Issue/提案 → 认领任务 → 提交 PR → 评审合并
- 开发规范与流程参考：[`docs/19-dev-guide.md`](19-dev-guide.md)
- 贡献计入 CP，详情见 [`docs/11-economics.md`](11-economics.md)

## 7. 经济与权益
- CP：贡献权益（不可转让）
- UC：使用权益（仅限应用商店内使用）
- 试用与增长权益：新用户权益包/试用包/成长任务
详情见 [`docs/11-economics.md`](11-economics.md)

## 10. 成员行为建议
- 先文档后实现，确保可复用与可传递
- 遵守隐私与安全原则
- 涉及云资源/域名/账号等资产时，需登记、按规则报销并协助交接

## 11. 协作约定（建议 + 底线）
**建议**
- 每周保持可见产出（文档或代码），并在 Issue 中沉淀结论
- 会议提前约定时间；如无法参加请提前说明
- PR/Issue 中写清影响范围与测试方式，便于评审

**底线**
- 涉及云资源、账号与权限必须登记并最小授权
- 关键变更需有可追溯记录（Issue/PR/文档）
- 对外发布内容需完成评审与口径确认

## 12. 第一次贡献建议
- 完成一条文档补充或术语修正
- 选择一个小型任务并在 2 周内交付
- 参与一次评审或共创展示
