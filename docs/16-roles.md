# 角色体系与协作分工

本文件用于定义社区内部角色与协作机制，确保文档与产品持续演进。

## 角色列表（初版）
- 文档总编：统一口径、结构、版本控制与导航
- 产品架构官：产品矩阵、路线图、里程碑、验收标准
- 技术标准官：身份/授权/数据结构/接口规范对齐
- 治理与运营官：提案机制、贡献流程、角色权限与激励
- 经济与权益官：权益体系、试用与定价、风控与合规边界
- 市场官：对外传播策略、品牌叙事与市场定位
- 增长官：用户增长、转化路径、试用策略与数据复盘
- 法务官：合规边界、风险评估与对外口径审核
- 生态合作官：合作策略、对外沟通、案例沉淀
- 质检与审校官：一致性、可读性、链接与术语校对
- 产品负责人（Product Owner）：单产品方向与落地推进

## 协作机制（建议）
1) 角色认领：每个产品至少 1 名产品负责人
2) 周期更新：每周更新里程碑与风险清单
3) 文档同步：产品变更必须同步到文档
4) 复盘机制：每个里程碑结束后复盘
