# 参与方式与贡献路径

我们欢迎开发者、研究者、创作者与投资方参与共建。以下是可持续参与的入口与方式。

## 贡献方向
- **开发者**：协议、工具链、插件、应用集成
- **研究者**：方法论、评估指标、路线验证
- **创作者**：内容、案例、产品叙事与社区传播
- **投资方/伙伴**：场景共建、资源支持、生态协同

## 参与路径（建议流程）
1. 阅读 `docs/00-overview.md` 了解整体方向
2. 选择兴趣方向（数据层/协议层/AI层/应用层）
3. 进入对应工作组或提出提案
4. 以任务形式交付成果并获得反馈

## 贡献流程（建议）
1. 在仓库中提交 Issue 或讨论提案，说明目标与范围
2. 认领任务或创建分支，明确验收标准与时间计划
3. 提交 PR（或文档变更），附测试说明与影响范围
4. 评审通过后合并，记录贡献与 CP
5. 进入路线图或发布说明（如适用）

## 评审与合并标准
- 功能完整、可复现，关键路径具备基础测试
- 文档与代码同步更新
- 风险可控（安全/隐私/性能）
- 具备复用价值或可推广到其他模块

## 贡献奖励（与 CP 绑定）
- 贡献通过后计入贡献权益（CP）
- CP 可提升提案优先级与资源倾斜
- 可获得使用权益（UC）奖励或服务抵扣（按经济模型规则）
详情见 `docs/11-economics.md`。

## 贡献原则
- 尊重用户主权与隐私优先
- 产出可验证、可复用的成果
- 文档与代码同步，保证可持续协作

## 新手入口
- 文档改进与术语校对
- 小型工具或脚手架
- 示例应用与教程
