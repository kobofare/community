# 云资源与资产管理（参考版）

> 目标：统一云资源申请、登记、使用与交接流程，避免资源流失与权限失控。

## 1. 适用范围
- 社区内涉及云主机、域名、对象存储、账号/密钥等资产
- 适用于开发/测试/生产环境

## 2. 资产类型
- 云主机（ECS/VM）
- 域名与证书
- 对象存储/数据库/缓存
- 账号与访问凭证（API Key/Token）

## 3. 申请与登记流程（建议）
1) 提出需求（用途/预算/时长/负责人）
2) 管理员确认与审批
3) 采购或开通资源
4) 登记资产信息（见“登记字段”）
5) 分配访问权限（优先密钥/最小权限）

## 4. 报销与成本
- 按社区规则与项目预算执行
- 报销需提供订单/账单与用途说明
- 若涉及多人协作，需明确成本归属

## 5. 到期与告警
- 负责人需关注到期/余额告警
- 出现告警时，及时通知管理员
- 续费、降配或释放需形成记录

## 6. 退出与交接
- 退出或更换负责人时，需完成资源交接
- 交接包括：账号归属、密钥更新、部署说明
- 社区采购资产需在到期前继续支持社区使用

## 7. 访问与安全
- 禁止 root 密码登录，优先使用 SSH 密钥
- 权限最小化，禁止共享个人凭证
- 重要变更应有变更记录或备注

## 8. 资产登记字段（建议）
- 资产名称/用途
- 项目/仓库
- 环境（dev/test/prod）
- 服务商/地域
- 资源 ID / 域名
- 负责人/协作者
- 创建时间/到期时间
- 费用与支付方式
- 访问方式（密钥/账号）
- 备注（部署说明/限制）
