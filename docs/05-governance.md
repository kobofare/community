# 协作与治理机制

我们坚持开源共建与透明治理，确保方向、规则与成果可追踪、可复用、可迭代。

## 治理原则
- 公开透明：决策与讨论可追溯
- 贡献导向：以成果与贡献为主要评价
- 协作优先：鼓励跨角色的协同推进
- 可持续：避免短期主义对长期建设的干扰

## DAO 方向（渐进式）
我们认可需要领导与制衡并存：既要激励有远见的优秀领导者，也要限制权力无序扩张，避免偏离社区共识。治理将以提案、任务与贡献为基础，并在合规前提下**渐进式**推进社区共治。  
DAO 方向与路线详见 [`docs/21-dao-direction.md`](21-dao-direction.md)。

## 协作方式
- **提案机制**：重要方向与规则通过提案讨论与共识达成
- **任务驱动**：将目标拆解为明确可交付的任务
- **工作组**：围绕关键方向建立专项工作组

## 协作约定（建议 + 底线）
> 详细参考：[`docs/14-member-handbook.md`](14-member-handbook.md)、[`docs/19-dev-guide.md`](19-dev-guide.md)、[`docs/20-cloud-assets.md`](20-cloud-assets.md)

**建议**
- 每周保持可见产出（文档或代码），并在 Issue 中沉淀结论
- 会议提前约定时间；无法参加请提前说明
- PR/Issue 中写清影响范围与测试方式，提升评审效率

**底线**
- 代码与关键变更必须可追溯（Issue/PR/文档）
- 云资源与账号权限必须登记并最小授权
- 对外发布内容需完成评审与口径确认

## 提案模板（建议）
- 标题 / 背景 / 目标
- 方案与范围（包含不做的部分）
- 资源需求（人力/预算/依赖）
- 里程碑与验收标准
- 风险与回滚计划
- 负责人/工作组

## 决策流程（建议）
1) 提案提交与公开讨论
2) 维护者/评审组初审
3) 社区评审或投票（按影响级别）
4) 执行与验收
5) 复盘与文档归档

## 角色更替与退出机制
- 角色可由工作组协商更替
- 关键岗位需完成交接与文档说明

## 角色与职责（示例）
- 维护者：把控质量与方向，合并关键变更
- 贡献者：提交代码、文档、研究与产品实践
- 研究者：提供方法论、评估与路线建议
- 生态伙伴：提供资源、场景与合作机会

## 透明与记录
- 关键讨论与决策形成文档
- 里程碑与成果公开同步
- 重要规则与流程持续迭代
