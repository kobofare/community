# 路线图

本路线图以 2026-02-06 为起点，聚焦 90 天与中长期目标。内容会随社区进展持续更新。

## 近期目标（2026-02-06 至 2026-05-07）
- 统一数据主权叙事与技术路线的文档体系
- 夯实钱包插件与授权流程的用户体验
- 扩展数字人场景与能力组件
- 形成 DAO 渐进路线与治理规则草案
- 完成一次公开共创与可展示成果

| 目标 | 负责人/工作组 | 验收标准 |
| --- | --- | --- |

## 中期目标（未来 1 年）
- 形成稳定的授权与治理协议规范
- 建立可复用的应用集成标准
- 形成可持续的协作机制与贡献体系

## 优先合作场景（阶段聚焦）
- 跨境贸易
- 社交与分发
- 个人知识库
- 个人工具与生产力

## 长期愿景（未来 2-3 年）
- 打造用户可控的数据资产与 AI 协作生态
- 形成跨平台、可迁移的个人智能体标准
- 让社区成为数据主权与个人智能体的公共基础设施

## 2026 方向草案（来自历史规划，已结合现状调整）
### 商业探索
- 面向线下商户与跨境贸易场景，提供可验证授权与可定制的渠道能力
- 构建“资金/权益/工具”的应用内循环，形成权益兑换与结算机制（仅限应用商店内部，详见 [`docs/11-economics.md`](11-economics.md)）

### 产品方向
- 打造应用商店脚手架，支持 DApp 快速上架与钱包/存储/合约自动对接
- 打磨数字人、智能助手与资产管理工具，形成可复用能力组件
- 加固基础设施并推动更多节点建设（区块链网络/存储网络等）

## 探索任务（可执行，持续校准）
- 完成钱包插件 + 数字仓库的端到端授权联调，形成可演示的用户数据主权闭环
- 输出最小可用的授权与治理规则草案（数据目录、授权范围、撤销机制）
- 将面试官/督学老师数字人抽象为可复用能力组件，发布统一 API 与集成示例
- 让知识库完成一次真实业务接入示范（应用接入、摄取、检索与记忆闭环）
- 打造生产力工作台 PoC（文本/语音/图像任选一条完整链路可用）
- 试运行 DAO 工作组机制（提案分级、评审、投票与复盘）
- 搭建分散式社交最小节点（基于开放协议），跑通身份与内容发布流程
- 实现分散式计算的基础任务流程（统计/分析任务提交、执行与结果回传）
- 设计区块链游戏的轻量化原型，验证玩法与价值协作路径

### 探索任务与产品矩阵对应
- 钱包插件 + 数字仓库联调 → 数据主权基础设施
- 授权与治理规则草案 → 协议层规范
- 数字人能力组件化 → 数字人应用矩阵
- 知识库业务接入 → 知识库中台
- 生产力工作台 PoC → 超级 APP 方向验证
- 分散式社交/计算/游戏 → 生态扩展与新场景
